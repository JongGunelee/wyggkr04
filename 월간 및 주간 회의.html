메모 및 미작성 데이터 관리

현재 HTML에 저장된 모든 데이터를 GitHub 관리용 memos.json과 skipped.json 파일로 다운로드합니다.
GitHub 모드로 처음 전환할 때 이 기능을 사용하세요.

내장 데이터 모드 사용법 및 작동 방식

현재 사용 중인 기본 방식입니다. 모든 메모와 미작성 날짜 데이터가 이 HTML 파일 안에 직접 저장됩니다.

  • 데이터 불러오기: 페이지 로딩 시, HTML 파일 내의 HARDCODED_MEETING_MEMOS 변수와 HARDCODED_SKIPPED_DATES 변수에서 데이터를 직접 읽어옵니다.
  • 데이터 저장하기: [모든 변경사항 저장] 버튼을 누르면, 현재 메모와 미작성 날짜 변경사항이 적용된 새로운 HTML 파일 전체를 다운로드합니다. 이 새 파일을 열어서 계속 사용하시면 됩니다.

GitHub 수동 업데이트 설정

GitHub 수동 업데이트 모드 사용법 및 작동 방식

데이터를 GitHub 저장소의 별도 JSON 파일로 관리하여 HTML 파일과 분리하는 방식입니다. 메모와 미작성 날짜 데이터는 항상 한 쌍으로 함께 처리됩니다.

  1. 준비: GitHub 저장소에 memos.json과 skipped.json 파일을 생성합니다. (내장 데이터를 사용했다면, 위의 [JSON 파일 생성] 버튼으로 파일을 먼저 만드세요.)
  2. 설정: 위 입력칸에 각 파일의 Raw URL을 붙여넣습니다. (GitHub에서 파일 보기 > 'Raw' 버튼 클릭 후 주소 복사)
  3. 데이터 불러오기: 페이지 로딩 시, 위에서 설정한 두 URL에서 각각 메모와 미작성 날짜 데이터를 가져와 화면에 표시합니다.
  4. 데이터 저장하기: [모든 변경사항 저장] 버튼을 누르면, 현재 변경된 메모와 미작성 날짜 데이터로 memos.json과 skipped.json 두 개의 파일을 다운로드합니다.
  5. 업데이트: 다운로드된 두 파일을 GitHub 저장소에 덮어쓰기(업로드)하고 commit하면 변경사항이 영구적으로 저장됩니다.

GitHub API 자동 업데이트 설정

GitHub API 모드 사용법 및 작동 방식 (고급)

⚠️ 경고: Personal Access Token(PAT)은 비밀번호와 같습니다. 노출되지 않도록 각별히 주의하세요. 이 방식 역시 메모와 미작성 날짜 데이터를 한 쌍으로 함께 처리합니다.

  1. 준비: GitHub 저장소에 memos.json과 skipped.json 파일을 미리 생성해 둡니다. (위의 [JSON 파일 생성] 버튼으로 파일을 먼저 만드세요.)
  2. 토큰 발급: GitHub에서 저장소 접근 권한(repo 스코프)이 있는 PAT를 발급받습니다.
  3. 설정: 위 설정값들을 모두 정확하게 입력합니다. (토큰은 페이지를 새로고침하면 보안을 위해 사라집니다)
  4. 데이터 불러오기: 수동 모드와 마찬가지로, GitHub 저장소의 파일 경로를 기반으로 메모와 미작성 날짜 데이터를 원격으로 불러옵니다.
  5. 데이터 저장하기: 변경사항 발생 2초 후 자동으로 저장됩니다. 또는 [모든 변경사항 저장] 버튼으로 즉시 강제 저장할 수 있습니다.
