081 공정 관리 앱 - 모드 전환기 & 오프라인 매뉴얼

오프라인(망분리) 구동 메커니즘 가이드

본 HTML 앱은 인터넷망이 완전히 차단된 에어갭(Air Gap) 또는 로컬(file://) 환경에서도 완벽히 동작하도록 설계되어 있습니다. 코드가 어떻게 이를 가능하게 하는지 상세한 배경 지식을 공유합니다.

1. Web Worker 보안 정책 자동 우회 (Main-Thread 폴백)
로컬 보안 정책에 따라 file:// 환경에서는 대용량 엑셀 데이터를 병렬 처리하는 Web Worker 객체의 생성이 차단됩니다. 하지만 앱은 이를 에러(Crash)로 인식하지 않고 내부 try...catch에서 포착하여, 메인 스레드 상의 window.XLSX 모드로 즉각 우회(Fallback)시킵니다. 즉, 인터넷 오프라인일 뿐만 아니라 로컬 PC에서도 무리 없이 엑셀 파일을 읽습니다.
2. 외부 API 동적 우회 처리 (Graceful Degradation)
GitHub 등에서 최신 파일을 받아오는 "자동 이력/업로드 기능"은 접속 실패를 앱 치명적 에러로 스택에 올리지 않습니다. 단순 showToast() 경고만 띄우며, 로컬의 .xlsb 파일을 버튼으로 직접 드래그&업로드할 수 있게 100% 수동 전환됩니다.
⚠️ 오프라인 사용 시의 3가지 제약 및 주의점
  • 대용량 파일 파싱 시 UI 일시정지: Web Worker가 차단되어 메인 스레드에서 즉각적으로 연산을 수행합니다. 이로 인해 50MB 이상의 파일 로드 시 3~5초 가량 화면이 멈출(Freezing) 수 있습니다. 정상 동작이므로 안심하십시오.
  • 의존성 파일 관리 필수: 오프라인 모드로 변환한 경우 반드시 HTML 파일과 동일한 위치에 libs 폴더(Tailwind, ExcelJS, SheetJS, 폰트 포함)가 세트로 존재해야 합니다. 단독 이동 시 스타일/기능이 작동하지 않습니다.
  • 자동 동기화 불능: GitHub 등 연결 차단으로 인해 "잔여 엑셀"을 자동으로 땡겨오는 기능이 마비됩니다, 파일을 수동으로 받아 입력해야 합니다.
