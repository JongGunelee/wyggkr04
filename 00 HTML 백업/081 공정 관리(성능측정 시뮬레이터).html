[🧠 081 데이터 적재 상관관계 정밀 가이드]
- 📋 기본 데이터 (프로젝트 워크시트): 시스템의 뼈대가 되는 메인 엑셀 시트입니다. 기본 렌더링 DOM 부하는 이 메인 시트의 누적 행(Row) 수와 고정 열(Column, 현재 17~20개) 수의 곱으로 1차 결정됩니다.

- 🔄 동적 부수 객체 (작업일, 마일스톤, 정산 등): '📌 작업일', '💎 마일스톤', '💰 정산', '⛓️ 의존성', '🔗 웹 링크' 시트는 메인 프로젝트에 종속되는 데이터입니다. 이 개별 시트들 내부의 열(Column) 개수는 프론트엔드 렌더링 성능 계산식에 직접 반영하지 않아도 무방합니다. (💡 이유: 프론트엔드 화면에 구현될 때 이 종속 데이터의 열 항목들은 독립적인 표(Table)의 셀(Cell)로 낱낱이 그려지지 않습니다. 대신 차트 상의 막대(Bar)나 아이콘 같은 '단일 화면 요소(DOM)의 내부 속성(Metadata)'으로 병합되어 처리됩니다. 즉, 숨겨진 텍스트 속성이 몇 개 늘어나는 것보다 화면에 실제로 그릴 물리적 도형(발생 건수) 자체가 늘어나는 것이 브라우저 렌더링 부하의 진짜 원인이기 때문입니다.)

- 📈 폭발적 증식의 상관관계: 성능 및 메모리 부하를 결정짓는 핵심 변수는 동적 워크시트 데이터의 '너비(열)'가 아니라, '발생 건수(행의 수)'입니다. 전체 시스템 부하는 [메인 프로젝트 행 수] × ([기본 고정 열 수] + [종속된 부수 객체의 총 발생 건수]) 공식을 따릅니다. 즉, 행이 하나 추가될 때마다 종속된 객체의 개수만큼 메모리 노드가 복리로 증식하므로 객체 밀도(개수) 통제가 가장 중요합니다.

🛠️ 모니터링 실행 시 아무 반응이 없다면? (유형별 해결 방법)

유형 1. 브라우저 팝업 차단

증상: 버튼을 눌러도 새 창이 뜨지 않음 (가장 흔함)

  1. 브라우저 주소창 우측 끝을 확인합니다.
  2. '팝업 차단됨' (창 모양에 ❌ 표시) 아이콘을 클릭합니다.
  3. "항상 팝업 허용"을 선택하고 완료를 누릅니다.
  4. 분석 시작 버튼을 다시 클릭하여 실행합니다.
유형 2. 샌드박스 보안 제약

증상: 팝업 알림도 없고 아예 실행이 안 됨

  1. 현재 보고 계신 온라인 미리보기(Canvas) 환경은 보안상 새 창 띄우기가 원천 차단되어 있을 수 있습니다.
  2. 전체 코드를 복사하여 내 PC에 test.html 등의 이름으로 저장합니다.
  3. 저장한 파일을 크롬에서 직접 더블클릭해 열고 다시 테스트합니다.
유형 3. 빈 화면 출력

증상: 새 탭은 열렸지만 하얀 화면만 나옴

  1. 업로드한 HTML 자체에 화면에 그릴 내용(시각적 요소)이 없는 경우 지극히 정상적인 현상입니다.
  2. 이 시뮬레이터 원본 창으로 돌아와서 우측 모니터링 수치(FPS, DOM 등)가 변하는지 확인합니다.
  3. 수치가 움직인다면 백그라운드 분석이 완벽히 작동 중입니다.

📖 시스템 사용 도움말 및 매뉴얼

📊 1. Track A: 데이터 구동 시뮬레이션

개발 초기 단계나 아키텍처 설계 단계에서 사용합니다. 실제 코드를 작성하기 전, 엑셀 데이터의 규모(행, 열)와 부가적인 의존성 객체(마일스톤, 링크 등)의 수량을 슬라이더로 조절하여 예상되는 브라우저의 렌더링 부하와 메모리 점유율을 수학적으로 예측합니다.

🔄 2. Track B: 실제 코드 모니터링

개발이 어느 정도 진행된 후 작성된 소스 코드를 검증할 때 사용합니다. 테스트할 HTML 파일을 화면에 드래그 앤 드롭으로 업로드하면, 가상 환경에서 코드를 안전하게 실행시키고 런타임 성능(FPS, Memory Heap, DOM 개수)을 실시간으로 추적하여 병목 구간을 탐지합니다.

🔖 3. Track C: 북마클릿 (오버레이 모니터)

실제 서버에 배포된 운영 환경이나 로컬 서버(Localhost) 환경에서 브라우저 위에 띄워놓고 사용할 수 있는 도구입니다. 제공되는 북마클릿 버튼을 브라우저의 즐겨찾기 바에 드래그하여 설치한 뒤, 어떤 웹 페이지에서든 클릭 한 번으로 성능 모니터링 UI를 소환할 수 있습니다.

📋 4. 종합 결과 리포트

Track A의 시뮬레이션 수치를 바탕으로 현재 프로젝트의 전체 시스템 부하 등급(S, A, B, C, D, F)을 자동으로 판정합니다. 계산된 객체 밀도를 견디기 위해 사용자 PC가 갖춰야 할 '최적/권장/최소 하드웨어 스펙' 가이드라인을 함께 제시하여 고객사 배포 기준을 세우는 데 도움을 줍니다.
