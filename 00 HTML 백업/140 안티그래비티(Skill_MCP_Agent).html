2. 무결성 및 보안 프로토콜 (Integrity & Security)

🚫 절대 업로드 금지 (Strict Block)

  • Environment Files: .env, config.js (Secrets 포함)
  • Key Files: .pem, .key, id_rsa, private_key
  • System/Build: node_modules, .DS_Store, Thumbs.db
  • Large Binary: .exe, .dll, .zip (LFS 미설정 시)

✅ 무결성 검증 표준 (Signature)

  • GPG Signing: 모든 커밋은 GPG 키로 서명되어야 함 (Verified 뱃지 획득)
  • Hash Check: 다운로드한 스킬/에이전트 파일의 SHA-256 해시 검증 필수
  • Dependency Scan: npm audit 및 Dependabot 상시 활성화

⚡ 옵션 2: PowerShell 원라인 (스킬+에이전트 통합 설치)

powershell -ExecutionPolicy Bypass -Command "mkdir -Force \"$env:USERPROFILE\.agent\skills\"; mkdir -Force \"$env:USERPROFILE\.agent\workflows\"; git clone https://github.com/sickn33/antigravity-awesome-skills.git \"$env:USERPROFILE\.agent\skills\"; git clone https://github.com/first-fluke/oh-my-ag.git \"$env:TEMP\oh-my-ag\"; Copy-Item -Recurse -Force \"$env:TEMP\oh-my-ag\.agent\workflows\*\" \"$env:USERPROFILE\.agent\workflows\"; [Environment]::SetEnvironmentVariable('ANTIGRAVITY_SKILLS_PATH', \"$env:USERPROFILE\.agent\skills\", 'User'); Write-Host '✅ 설치 완료!' -ForegroundColor Green"

# ===== 폴더별 용량 확인 스크립트 =====
Write-Host "`n💾 폴더별 용량 분석...`n" -ForegroundColor Cyan

$folders = @(
  "C:\Users\$env:USERNAME\.agent",
  "C:\Users\$env:USERNAME\.gemini",
  "C:\Users\$env:USERNAME\.claude",
  "C:\Users\$env:USERNAME\AppData\Local\Temp\antigravity",
  ".\140_engine.js"
)

foreach ($folder in $folders) {
  if (Test-Path $folder) {
    $size = (Get-ChildItem $folder -Recurse -File -ErrorAction SilentlyContinue |
      Measure-Object -Property Length -Sum).Sum
    $sizeMB = [math]::Round($size / 1MB, 2)
    $color = if ($sizeMB -gt 500) { "Red" } elseif ($sizeMB -gt 100) { "Yellow" } else { "Green" }
    Write-Host " $folder : ${sizeMB} MB" -ForegroundColor $color
  }
}

🔣 암호 같은 기호의 의미
  • %...% (CMD 변수): 윈도우가 미리 정해둔 주소창 별명 (예: %USERPROFILE% = 내 문서 폴더)
  • $env: (PS 변수): 파워쉘(PowerShell) 환경에서 사용하는 주소창 별명
  • .agent (점 폴더): 시스템용 숨김 폴더나 설정 폴더를 뜻하는 관례
  • @ (호출 기호): 특정 스킬이나 에이전트를 지목하여 '소환'할 때 사용
  • $ (변수/명령): 프로그래밍에서 값을 담는 상자나 터미널 명령줄을 의미
📍 주소(경로) 읽는 법
  • 절대 경로: C:\Users\John\... 처럼 뿌리(C드라이브)부터 끝까지 다 적은 전체 주소
  • 상대 경로: .\skills 처럼 '지금 내 위치'를 기준으로 적은 단축 주소
  • / vs \ : 인터넷·리눅스는 /을, 윈도우는 \를 구분자로 쓰지만 최근 AI는 둘 다 잘 알아듣습니다.
  • 패스(Path): 폴더가 겹겹이 쌓여있는 '길'을 의미합니다.

🔍 AI와 개발자의 천안통: Ripgrep (rg)의 기원과 위력

안티그래비티 에이전트들이 사용하는 rg (또는 ripgrep) 명령어는 단순히 글자를 찾는 도구가 아닙니다. 이는 Andrew Gallant(BurntSushi)라는 천재적인 개발자가 Rust 언어로 개발한 현대 검색 공학의 결정체입니다.

🛠️ 개발 배경 및 철학

고전적인 grep의 한계를 넘어, 수백만 줄의 대규모 코드베이스를 지연 없이 검색하기 위해 탄생했습니다. VS Code의 공식 검색 엔진으로 사용될 만큼 신뢰성과 속도가 검증되었습니다.

⚡ 압도적인 성능의 비밀

CPU의 SIMD(벡터 연산) 기술을 극대화하고, .gitignore 파일을 지능적으로 분석하여 불필요한 검색을 원천 차단합니다. 0.1초 만에 기가바이트 단위의 파일을 읽어냅니다.

🤖 AI 에이전트와의 궁합

에이전트들이 복잡한 버그를 찾을 때 rg를 쓰는 이유는 "맥락 파악의 속도" 때문입니다. AI가 당신의 전체 프로젝트를 순식간에 훑어보고 답변할 수 있게 해주는 핵심 엔진입니다.

"ripgrep is a line-oriented search tool that recursively searches your current directory for a regex pattern. By default, ripgrep will respect your .gitignore and automatically skip hidden files/directories and binary files."

💡 초보자를 위한 상식: 왜 JSON을 쓰나요?

🤔 JSON이 뭔가요?

JSON(JavaScript Object Notation)은 "컴퓨터와 사람이 모두 읽기 편한 데이터 약속"입니다.
마치 택배 송장처럼 "이름": "홍길동" 형식으로 정보를 명확하게 정리한 것입니다.

🎯 왜 필요한가요?

복잡한 정보(예: 내 친구 목록, 그 친구의 취미들)를 구조화해서 오차 없이 전달하기 위해서입니다. 엑셀(CSV)로는 이런 '입체적인' 정보를 담기 어렵습니다.

    🏆 안티그래비티(AG) IDE 표준 실전 활용 마스터 클래스 (E2E Workflow)

    아이디어에서 실제 배포까지, 모든 요원과 도구를 동원한 프로젝트 실전 흐름입니다.

    ⚖️ 안티그래비티 도입 전/후 비교 분석 (Power of Ecosystem)

    🧠 Skill (지능 계층)

    ❌ 미적용 (Without Skill)
    기본 LLM의 일반적인 답변에만 의존. 대량의 수식을 수동으로 검증해야 하며, 매번 수천 줄의 가이드라인을 프롬프트에 붙여넣어야 함.
    ✅ 적용 (With Skill)
    검증된 'Logic Pack'이 즉시 주입되어 복잡한 연산 무결성을 100% 보장. 프롬프트 오염 없이 프로젝트 특화 지식(Skill)을 AI가 즉각 활용.

    🔌 MCP (연결 계층)

    ❌ 미적용 (Without MCP)
    AI가 로컬 시스템을 보지 못하여 사용자가 수천 개의 파일을 직접 읽어주거나 복사해서 대화창에 올려야 함. 명령어 수행 권한이 없어 수동 작업 반복.
    ✅ 적용 (With MCP)
    AI가 보안 샌드박스 내에서 파일 시스템, 터미널, 브라우저를 직접 통제. 수천 개 파일의 일괄 검색-수정-테스트를 AI가 자율적으로 수행.

    🤖 Agent (수행 계층)

    ❌ 미적용 (Without Agent)
    단순 질문-응답(Q&A) 방식. 전체 프로젝트의 방향성을 상실하거나, 오류 발생 시 사용자에게 일일이 해결법을 물어보며 작업이 중단됨.
    ✅ 적용 (With Agent)
    오케스트레이터 기반 자율 수행. 목표(Goal)만 주어지면 오류 발생 시 스스로 원인을 분석하고 코드를 수정하며 프로젝트를 최종 완성.

    GSD(Get Shit Done)란?
    AI와의 긴 대화로 인해 발생하는 '컨텍스트 저하(Context Rot)'를 방지하기 위해 작업을 Discuss(논의) → Plan(기획) → Execute(실행) → Verify(검증) 단계로 완전히 분리하여 수행하는 프로급 AI 개발 방법론입니다.

    💡 0% -> 100% 마스터 가이드

    • Step 1 (Discuss): 구현할 기능의 요건을 설명하고 AI와 브레인스토밍합니다. (결과물: PROJECT.md)
    • Step 2 (Plan): 구체적인 구현 단계와 기술 스택을 정의합니다. (결과물: PLAN.md)
    • Step 3 (Execute): Plan에 명시된 항목을 하나씩 원자적으로 코딩합니다. (결과물: 소스코드)
    • Step 4 (Verify): 테스트 및 브라우저 검증을 통해 기능 완성을 확정합니다. (결과물: STATE.md)
